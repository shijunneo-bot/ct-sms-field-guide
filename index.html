This is a pattern library, not a template dump. Every card is a job SMS is genuinely good at, paired with the message, the trigger that makes it land, and whether it will deliver in your market. Here is how to turn a card into a live send.

1Find the fit

Filter by your vertical (the colour), or switch Group by to Vertical to browse a whole industry. Know the job already? Search a word: try “renewal”, “OTP”, or “abandoned”.

2Check the trigger

Each card leads with a timing signal: the event that makes the message relevant right now. If that signal is not live for your audience, it is not the right send yet.

3Confirm it delivers

Read the “Delivers in market” flags. Green: send as written. Amber: needs a registered Sender ID or a local rule. Red: the copy needs a market variant, usually link-free. Open the card for the compliant version.

4Adapt the copy

Hit the copy button on any message to grab the exact text, then swap [Brand] for your sender name, localise, and keep to one segment. An emoji flips SMS to Unicode at 70 characters per segment, so it costs more.

Badge = the use case Colour = the vertical delivers as written needs a rule / registration needs a market variant Pattern = one-off / recurring / automated / journey

SMS earns its place on reachability, speed, and no-app reach. Where WhatsApp or push already reaches your user well, lead with those and keep SMS for the moments only it wins. Each card’s “Why SMS, not WhatsApp” line makes that call for you.

ImplementBuild it in CleverTap▾

Found a pattern you like? Here is how it maps to the CleverTap SMS campaign builder, and what you can personalise once the copy is in.

The five steps
1Start here

Pick how users qualify: past behaviour or a custom list, live behaviour in real time, or an external API trigger. Confirm your SMS provider is connected, and optionally switch on conversion tracking.

2Who

Choose the target segment, or All Users, add rules, set a control group, and calculate estimated reach.

3What

Write the message. Paste a card’s copy here, then swap the placeholder for a personalisation token. The editor counts characters and SMS segments live.

4When

Send now, schedule for later, or set recurring, in your chosen timezone. Triggered and journey patterns take their timing from the trigger or the Journey.

5Publish

When every step shows green, publish the campaign.

The one thing that blocks everyone: the “What” step stays locked until an SMS service provider is connected to the account (Settings, then SMS). Until then you will see “No SMS Service Provider set.” That connection is where your Sender ID and the market’s delivery rules live, which is exactly what the compliance flag on each card is about.

Personalise the copy

The editor offers two token styles, from the {()} and @ buttons. They do the same job:

Name{{ Profile.name | default: "there" }}or  @Profile - Name
Phone{{ Profile.Phone | default: "" }}or  @Profile - Phone

So a line like “Hi [name], your order shipped” becomes “Hi {{ Profile.name | default: "there" }}, your order shipped.” Always set a default so the message still reads if the value is missing. Beyond Profile you can pull Event properties, live API values (Linked Content), Recommendations, and Catalog items. There is also an AI assist, the sparkle, to help draft or tighten a line.

What you can layer on
Live data

Linked Content pulls real-time values from your API at send time, like a balance, an ETA, or a price. This is what makes the “live” cards feel live.

Recommendations & Catalog

Personalised product or content picks, chosen per user.

Conditional tags

Show different copy depending on a user condition, inside one message.

Click tracking

Shorten URLs to measure taps. Check the card’s flag first, since some markets, Malaysia A2P for one, block links.
